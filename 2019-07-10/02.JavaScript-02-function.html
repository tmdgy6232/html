이벤트 헨들러

만약 버튼을 클릭하는 동작을 브라우저가 감시하고 있는 상황이 되면 버튼의 클릭 동작을 수시로 살펴야하는 불편한 상황이 발생을 할것이다. 이런 상황을 우리는 blocking이라고 하며 윈도의 이전에 운영체제, 콘솔환경의 어플리케이션에서는 가능한 일이나. 윈도우처럼 멀티테스킹, 멀티 스레드 환경에서는 절대 해서는 안되는 일이다.

그래서 운영체제 차원에서 이벤트라는 것을 지원하며 아래러머 버튼에 온클릭이라는 속성을 지정해주먼 운영체제가 해당버튼의 클릭여부를 감시하고 있다가 클린 동작이 발생하면 설정된 함수를 시랳ㅇ해준다.

이벤트가 발생했을 때 동작하는 함수, 객체, 이런것들을 이벤트 핸들러라고 한다.
